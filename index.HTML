Level 2 – Password Policy
From the onboarding printout, players find the dashboard link and implicit username j.smith. They log in using the weak password from Level 1, and the system forces them to create a stronger password that meets policy rules before continuing.
Password policy warning
Your credentials are insecure. To continue, create a stronger password that:
  • must be at least 14 characters long
  • must contain at least one uppercase letter
  • must contain at least one lowercase letter
  • must contains at least one number
  • must contains at least one special character(e.g. ., !, _, $, %)
  • cannot contain the username
  • cannot repeat the same character or number more than twice
  • cannot contain common words such as "password" or "admin"
Level 3 – Hidden Admin Panel
On the dashboard there is no visible admin tab. The admin-team message hints that admins can go straight to the admin page from the address bar. Players should discover the hidden page by adding /admin. In this portal, use the button below to simulate that step.
Admin Team
Just a reminder that the admin panel isn’t listed on the dashboard menu for security reasons. Admins can still access it directly if needed. If you already have the dashboard open, the quickest way is to go straight to the admin page from the address bar.
Mentor message thread
A colleague mentions that the admin password is often shared through an image reference, and that sometimes it helps to look a little deeper than what’s visible. This points toward the stego step used in Level 4.
Level 4 – Admin Password via Steganography
The materials include an image that contains hidden data. The image has been encoded using steghide and no passphrase is required. Your task is to extract the hidden file from the image. The extracted file contains the admin password needed to unlock the admin panel. Use the appropriate steganography tool to reveal the hidden content and then enter the discovered password below.
